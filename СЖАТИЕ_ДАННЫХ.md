## 📊 Характеристики

**Алгоритм:** zstd (Zstandard by Facebook)
- Быстрее gzip при схожем коэффициенте сжатия
- Настраиваемые уровни: 1 (быстро) - 22 (максимальное сжатие)
- Рекомендуемый по умолчанию: 3

**Кодирование:** base64
- Сжатые данные кодируются в base64 для XML-безопасности

**Порог сжатия:** 1 КБ (настраиваемый)
- Данные < 1 КБ не сжимаются (overhead больше выгоды)

**Многопоточность:** до 4 ядер
- Параллельное сжатие/распаковка для больших пакетов

**Формат хранения:**
- Все строки объединяются и сжимаются в один blob
- Сохраняется как один `<R>` элемент с атрибутом `compression="zstd"`

## 🔄 Workflow с сжатием

### Export со сжатием:
```
1. Читаем данные из БД
2. Конвертируем в строки (pipe-delimited)
3. Проверяем размер > минимального порога
4. Объединяем все строки
5. Сжимаем zstd (уровень из config)
6. Кодируем в base64
7. Сохраняем в <R> с атрибутом compression="zstd"
```

### Import со сжатием:
```
1. Парсим XML пакет
2. Проверяем атрибут compression="zstd"
3. Декодируем base64
4. Распаковываем zstd
5. Разбиваем на строки
6. Парсим каждую строку
7. Импортируем в БД
```

## 💡 Преимущества

**1. Экономия трафика:**
- TDTP пакеты могут быть большими (до 3.8 MB)
- Zstd дает ~3-5x сжатие текстовых данных
- Критично для сетевых передач

**2. Производительность:**
- Zstd быстрее gzip на декомпрессии
- Настраиваемые уровни для баланса скорость/размер
- Многопоточность для больших пакетов

**3. Совместимость:**
- Прозрачная работа: старые клиенты игнорируют сжатые пакеты
- Опциональная функция (можно отключить)
- Автоматическая распаковка при парсинге

**4. Интеграция:**
- Работает со всеми адаптерами
- Совместимо с XLSX экспортом
- Поддержка в processors pipeline
