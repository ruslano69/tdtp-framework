# Сайт-Ассистент для Инженера-Трансформаторщика — План платформы

> Дата: 2026-06-03  
> Статус: концепция, ожидает отдельного репозитория

---

## Идея

Сайт-ассистент — не лендинг и не форма. Это агентная платформа специалиста:
справочник, калькулятор, приёмник данных и генератор страниц в одном интерфейсе.

Первая вертикаль: **проектирование силовых трансформаторов большой мощности**.

Причины выбора вертикали:
- Чёткая логика, проверенные теоремы и законы — нет галлюцинаций
- Существующие инструменты: Фортран 80-90х, RALE (до 10 МВА), TDS (заброшен 2017)
- Ниша 10–600 МВА практически пуста
- Производственные данные закрыты у крупных игроков — возможность калибровки
- Рынок $38+ млрд к 2030, дефицит инженеров-конструкторов нарастает

## Схема базы данных (ядро знаний)

```sql
-- Концепты (атомарные единицы знания)
concepts     id, domain, category, symbol,
             labels:jsonb,       -- {ru: "...", en: "...", de: "..."}
             description:jsonb

-- Формулы (параметрическая модель = строка здесь)
formulas     id, concept_id,
             expression,         -- SymPy-совместимое выражение
             inputs:jsonb,       -- [{symbol, unit, type}, ...]
             outputs:jsonb,
             source_standard,    -- "IEC 60076-1" / "ГОСТ 3484"
             source_clause,      -- "п. 5.3.2"
             verified:bool

-- Параметры
parameters   id, formula_id, symbol, unit, type, min, max, default

-- Модели (граф зависимостей = набор формул + порядок)
models       id, name, domain,
             dag:jsonb           -- граф зависимостей, параметрическая модель

-- Нормативы
standards    id, code, edition, language, domain

-- UI-блоки (библиотека визуалов)
ui_blocks    id, type,
             schema:jsonb,       -- структура входов/выходов
             template:jsonb      -- шаблон рендера

-- Сессии (основа самообучения)
sessions     id, user_id, queries:jsonb, gaps:jsonb, model_used
```

**Главный принцип:** знание живёт в БД. Код только исполняет.
Добавить новую формулу = добавить строку, не деплоить.

## Внутренний язык модели (DSL / онтология)

Все концепты хранятся в языконезависимом виде:

```yaml
concept: BEAM_SIMPLY_SUPPORTED
  domain: structural_mechanics
  properties:
    L: { type: Length,         unit: m,   symbol: L }
    q: { type: LinearLoad,     unit: N/m, symbol: q }
    E: { type: ElasticModulus, unit: Pa,  symbol: E }
    I: { type: MomentOfInertia,unit: m⁴,  symbol: I }
  derived:
    M_max: { expr: "q*L**2/8",         label: {ru: "Макс. момент", en: "Max moment"} }
    f_max: { expr: "5*q*L**4/384/E/I", label: {ru: "Прогиб",      en: "Deflection"} }
  source: { ru: "СП 20.13330", intl: "ISO 2394" }
```

Правила мультиязычности:
- Внутренние ID всегда на английском (`concept_id: CORE_FLUX_DENSITY`)
- Отображение — через `labels:jsonb` на языке пользователя
- Поиск — через multilingual-e5-large эмбеддинги (pgvector)
- Числа и единицы — через Pint + locale
- Топонимы и имена — через стандартизированные ID, не строки

## Агентная архитектура

```
Пользователь (любой язык)
        ↓
   Intake Agent       — квалифицирует запрос, собирает контекст
        ↓
   Search Agent       — ищет концепты и формулы в БД (3 уровня иерархии)
        ↓
   Calc Agent         — исполняет DAG-модель через SymPy
   Doc Agent          — генерирует документ / форму
   Page Agent         — собирает страницу из UI-блоков
        ↓
   Memory Agent       — сохраняет контекст сессии
```

## Поэтапное проектирование

### Этап 1 — Ядро знаний
- Схема БД + API CRUD
- DSL для концептов и формул (YAML → БД)
- Загрузчик знаний
- Первые 30–50 концептов: трансформаторостроение
- Мультиязычность с нуля: ru / en

### Этап 2 — Расчётный движок
- SymPy исполняет формулы из БД (не из кода)
- Pint валидирует единицы автоматически
- DAG-решатель: каскадный пересчёт при изменении параметра
- API: `POST /calculate {model_id, params}` → results

### Этап 3 — Поиск и навигация
- pgvector эмбеддинги для всех концептов
- Трёхуровневый иерархический поиск
- Мультиязычная модель (multilingual-e5-large)
- Семантический поиск: "потери холостого хода" = "no-load losses" = "Leerlaufverluste"

### Этап 4 — AI-агент
- LLM как интерфейс к поисковику и движку
- Агент квалифицирует запрос → выбирает модель → считает
- Контекст сессии: помнит историю запросов
- Трассировка: "почему такой результат" → цепочка формул + источники

### Этап 5 — Библиотека визуалов
- Компоненты: CalcBlock, FormulaCard, ParameterSweep, ResultSummary, StepForm
- Страница генерируется агентом из блоков по запросу
- KaTeX для формул, D3.js для графиков, Three.js для 3D-моделей

### Этап 6 — Самообучение
- Логируем: что спрашивают, на что система не смогла ответить (`gaps`)
- Dashboard для эксперта: незакрытые вопросы → он докладывает базу
- Производственные данные → калибровка моделей (Этап после получения доступа)
- A/B результатов: какой ответ пользователь принял/отверг

## Стратегия убеждения технического директора

**Его позиция:** "AI не знает то, что знаю я"

**Ответ:** не спорить — показать что система делает то, что Фортран **не может**:

| Фортран 80-х | Платформа |
|---|---|
| Фиксированный расчёт | Параметрический DAG |
| Меняешь → пересчитываешь вручную | Меняешь → всё обновляется каскадно |
| Нет трассировки | Каждая формула = ссылка на ГОСТ/IEC |
| Один вариант | Параметрический свип → оптимум |
| Нет визуализации | Графики, сечения, зависимости |
| Один человек знает | База знаний доступна всей команде |

**Ключевой аргумент:** "Твои формулы уже в системе — с твоими ссылками,
твоими ограничениями. Ты не заменён — ты тиражирован."

**Дорожка к производственным данным:**
1. Демо на открытых расчётах → убеждаем ТД
2. Загружаем исторические ТУ и протоколы испытаний
3. Калибруем модели на реальных данных завода
4. Платформа знает "как у нас принято" — экспертиза ТД в системе

## Рыночный контекст

Исследование подтвердило:
- **RALE** (Швейцария): CHF 3 600/год, ограничен 10 МВА
- **JMAG**: $15 000–30 000/год, FEA-симулятор, не конструктор
- **ANSYS/COMSOL**: $30 000–80 000/год, общего назначения
- **TDS** (SourceForge): заброшен в 2017, 3/5 звёзд
- **AI-ассистент для проектирования трансформаторов**: не существует нигде

Ниша 10–600 МВА практически пуста. Производители в СНГ, Индии,
Юго-Восточной Азии работают на самописных Фортран-программах или Excel.
